O que é?

  • Superset/superconjunto de Javascript
  • Criado em 2012 pela Microsoft
  • Influências: Java, Javascript, C#

Vantagens

  • autocomplete (IDEs integram bem)
  • compilada: feedback de erros de forma rápida
  • possibilidade de integrar com javascript de forma simples e fácil
  • todo código javascript é um código typescript válido

Desvantagens

  • código mais verboso
  • curva de aprendizado maior (comparando com javascript)

Como funciona?

  • adiciona uma camada de verificação a mais no código
  • antecipa problemas que ocorreriam em runtime para tempo de compilação

Exemplo de código

Javascript

			  
				function my_sum(a, b) {
				  return a + b;
				}				
				
				let a = 4; 
				let b = "5";
								
				my_sum(a, b); //result: 45

Typescript

			  
				function my_sum(a: number, b: number) {
				  return a + b;
				}				
				
				let a = 4; 
				let b = "5";
								
				my_sum(a, b);
			  
			  9: Argument of type 'string' is not assignable to parameter of type 'number'.

Quando usar Typescript?

  • projetos grandes e complexos
  • grande quantidade de arquivos
  • equipes com vários desenvolvedores

  • arquivos de definição (.d.ts)
  • Ex: @types/react
  • DefinitelyTyped

Dúvidas comuns

  • Typescript vai substituir o javascript?
  • Tenho que adicionar tipo em tudo?

E como integrar com um projeto existente em JS?

  • Atualmente diversas dependências de terceiros (npm) já são escritas em Typescript
  • Typescript deve ser transpilado juntamente com o código javascript

Estratégia de Migração

Migrar todo o código existente

  • Problema: Muito tempo gasto sem novas funcionalidades

Migrar usando ferramentas (ex: airbnb/ts-migrate)

  • Problema: A maior parte do código não é migrada corretamente, sendo necessário rever quase tudo

Migrar aos poucos

  • Problema: Conviver com base de código dividida (JS e TS)
  • Código novo passa a ser criado usando TS
  • Código base (se algum) é migrado antes, pois será base para novos desenvolvimentos
  • Código antigo é migrado sob demanda, sempre que determinado arquivo precisar de alterações
  • Toda a equipe vai aprendendo junto, aos poucos
